# Contributing Guidelines

感谢你为 YarnProduction 项目做出贡献。请在提交代码前遵循以下规范。

## 1. 通用要求

- 所有沟通与说明文档优先使用中文。
- 代码需与现有项目结构保持一致，避免无关重构。
- 提交前确保项目可构建、关键流程可运行。

## 2. 函数编写规范

所有新增或修改的函数必须满足以下要求：

1. 添加清晰、描述性的注释文档。
   - JavaScript/TypeScript 使用 JSDoc。
   - C# 使用 XML 文档注释（`/// <summary>` 等），语义等价于 JSDoc。
2. 包含输入校验（参数空值、范围、格式等）。
3. 对错误条件使用早返回（early return），减少嵌套层级。
4. 使用有意义的变量名，避免 `tmp`、`data1` 等模糊命名。
5. 注释中至少包含一个示例用法（Example）。

## 3. C# 代码风格建议

- 优先使用异步方法与 `CancellationToken`。
- 在业务边界处进行参数校验并抛出明确异常。
- 对公共接口和核心实体补充完整注释。
- 对可空引用类型进行显式处理，避免潜在 `NullReferenceException`。

## 4. 提交与审查

- PR 描述应包含：变更目的、核心改动、验证方式。
- 若涉及接口变更，需说明兼容性影响。
- 若涉及数据结构或缓存键变更，需附带迁移说明。
